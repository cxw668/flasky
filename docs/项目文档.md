## 二、第 0 阶段：环境准备与项目启动（Day 1）

### 0.1 环境准备

- 创建虚拟环境：`python -m venv venv`
- 激活虚拟环境（Windows Git Bash）：`source venv/Scripts/activate`
- 安装依赖：`pip install -r requirements/dev.txt`
- 确认 Python 与 pip 指向虚拟环境

### 0.2 Flask 启动配置

- 设置环境变量：`FLASK_APP=flasky.py`
- 设置环境变量：`FLASK_CONFIG=development`
- 执行迁移：`flask db upgrade`
- 执行初始化部署：`flask deploy`
- 启动服务：`flask run`

### 0.3 启动验收

- 首页可访问
- 可注册新用户
- 可登录/退出
- 可发布一条帖子

## 三、第 1 阶段：理解 Flask 项目骨架（第 1 周）

### 1.1 入口与应用工厂

- 阅读 `flasky.py`，理解 CLI 命令注册（`test`/`profile`/`deploy`）
- 阅读 `app/__init__.py`，理解 `create_app()` 与扩展初始化顺序
- 说明为什么要用“应用工厂 + 蓝图”

### 1.2 配置系统

- 阅读 `config.py`，理解 `development/testing/production` 区别
- 找到分页配置项（`FLASKY_POSTS_PER_PAGE` 等）
- 修改分页数并观察页面变化

### 1.3 蓝图与请求流

- 阅读 `app/main/views.py` 中首页与用户页路由
- 阅读 `app/auth/views.py` 中登录与注册路由
- 阅读 `app/api/*`，识别 API 的组织方式
- 画出请求流：请求 → 路由 → 视图函数 → 模板/重定向/JSON

### 1.4 实操作业

- 在 `main` 蓝图新增 `/hello` 路由并返回模板或文本
- 在导航或首页增加该入口（可选）
- 本地手工测试新路由

### 1.5 阶段验收

- 能口头解释：应用工厂、蓝图、配置分层的作用
- 能独立新增一个页面并联调通过

---

## 四、第 2 阶段：数据库与用户系统（第 2 周）

### 2.1 数据模型理解

- 阅读 `app/models.py`：`User`、`Role`、`Post`、`Comment`、`Follow`
- 梳理关系：一对多、多对多（通过关联表）
- 总结 `Permission` 的位运算权限模型

### 2.2 认证与权限

- 理解 `UserMixin`、`login_manager.user_loader`
- 理解 `can()` 与 `is_administrator()` 的授权逻辑
- 理解 `@login_required` 与权限装饰器的使用场景

### 2.3 迁移实战（必做）

- 给 `User` 增加字段（例如 `website`）
- 生成迁移：`flask db migrate -m "add website to user"`
- 执行迁移：`flask db upgrade`
- 在个人资料页展示字段
- 手工验证：新增/编辑后数据持久化正常

### 2.4 阶段验收

- 能独立完成“改模型 → 迁移 → 页面展示”闭环
- 能解释为什么不能直接改库而跳过迁移

---

## 五、第 3 阶段：表单、模板、业务视图与 API（第 3 周）

### 3.1 表单与验证

- 阅读 `app/main/forms.py`、`app/auth/forms.py`
- 理解字段校验、错误提示、`validate_on_submit()`
- 给发帖或评论增加一个验证规则（如最小长度）

### 3.2 模板体系

- 阅读 `app/templates/base.html`，理解模板继承
- 阅读 `_macros.html`、`_posts.html`，理解模板复用
- 完成一个页面小改版（例如列表信息增强）

### 3.3 业务视图

- 跟读 `index()`、`post()`、`follow()`、`unfollow()`
- 理解分页逻辑与 cookie 控制“全部/关注”视图
- 理解 flash 消息与重定向时机

### 3.4 API 入门

- 阅读 `app/api/posts.py`、`users.py`、`comments.py`
- 理解认证方式与错误处理方式
- 给某个 API 返回增加一个字段
- 用 HTTPie/Postman 调试至少 2 个 API

### 3.5 阶段验收

- 能独立实现：表单规则 + 页面展示 + API 字段扩展
- 能说明页面渲染和 REST API 的协作关系

## 六、第 4 阶段：测试与部署认知（第 4 周）

### 4.1 测试基础

- 跑测试：`flask test`
- 阅读 `tests/test_basics.py`、`tests/test_user_model.py`
- 理解 `setUp/tearDown` 和测试数据库隔离

### 4.2 补充测试（必做）

- 新增 1 个模型测试（如密码校验、权限判断）
- 新增 1 个视图或认证流程测试（如登录重定向）
- 确保新增测试通过

### 4.3 部署认知

- 阅读 `boot.sh`，理解容器启动前的 `flask deploy`
- 阅读 `Dockerfile`、`docker-compose.yml`（了解即可）
- 了解开发/生产配置差异（`config.py`）

### 4.4 阶段验收

- 能说明：如何通过测试保障改动质量
- 能说明：服务启动时为什么要先迁移再启动

## 七、最终实战任务（综合）

> 在不破坏原有功能的前提下，完成一个“小功能闭环”。

- 需求定义：给用户资料新增“个人网站”字段

- 表单修改：编辑资料表单支持该字段
- 页面展示：用户主页显示该字段
- API 扩展：用户接口返回该字段（可选）
- 测试补充：至少 1 条相关测试
- 自测回归：注册、登录、发帖、评论流程均正常
